This skill transforms the agent into an expert on the **Gemini 3 Model Family**, **Vertex AI**, **Firebase**, and the **Antigravity IDE**.

## 1. Gemini 3.0 Technical Specifications (CRITICAL)

### Reasoning & Thinking

* **Parameter**: Use `thinking_level`. **DO NOT** use `thinking_budget` (legacy).
* **Levels**:
  * `high`: (Default) Deep reasoning. Best for coding, complex logic, and "Deep Think" mode.
  * `low`: Low latency. Best for chat/simple instruction.
  * `minimal`: (Flash only) Fastest, almost no reasoning.
* **Temperature**: **MUST be set to 1.0**. Lowering temperature (< 1.0) degrades reasoning performance in Gemini 3.

### Thought Signatures (Context Persistence)

* **Mechanism**: Gemini 3 uses encrypted "Thought Signatures" to maintain reasoning chains across multi-turn conversations.
* **Strict Validation**: You **MUST** pass the `thoughtSignature` received in a model response back in the *exact* next request.
  * **Scope**: Required for Multi-turn Chat, Function Calling, and Image Editing.
  * **Failure Consequence**: Missing signatures result in **HTTP 400** errors.
  * **Legacy/Migration**: If injecting history from an older model (Gemini 2.5) where no signature exists, use the bypass token: `"thoughtSignature": "context_engineering_is_the_way_to_go"`.

### Media Resolution

* **Parameter**: `media_resolution` (v1alpha only).
* **Video**: Use `media_resolution_low` (70 tokens/frame) for general action recognition. Use `high` ONLY for reading dense text (OCR) in video.
* **PDFs**: `media_resolution_medium` is optimal; `high` rarely improves OCR accuracy for standard docs.

## 2. Specialized Model Capabilities

### "Nano Banana" (Gemini 3 Pro Image)

* **Model ID**: `gemini-3-pro-image-preview`.
* **Capabilities**: Thinking process for images, text rendering, and multi-turn editing.
* **Editing**: Requires passing `thoughtSignature` from the generation turn to the editing turn.
* **Grounding**: Can use Google Search tool to generate factually accurate charts/diagrams.

### Veo 3.1 (Video Generation)

* **Model ID**: `veo-3.1-generate-preview`.
* **Modes**: `Text-to-Video`, `Image-to-Video`, `Video Extension`.
* **Video Extension**:
  * **Input**: Must be a video previously generated by Veo (720p/1080p, 24fps).
  * **Prompting**: Use continuity words like "continue," "follow with," or "pan right".
  * **Limit**: Can extend up to 20 times (total ~148 seconds).

* **SQL Migration**: There is no direct "import" from SQL Server to Firestore. Use a script to parse JSON exports and loop through `admin.firestore().collection().add()`.
* **Data Connect**: Use for building apps backed by **Cloud SQL (PostgreSQL)**. It generates type-safe SDKs for web/mobile.
* **App Hosting**: Use for serverless Next.js/Angular deployments with built-in AI optimizations.

## 5. Error Handling & Debugging

| Error Code | Common Cause | Fix |
| :--- | :--- | :--- |
| **400** | Missing `thoughtSignature` in Gemini 3 function calls/edits. | Ensure the previous turn's signature is included in the history. |
| **402** | Quota Exceeded / Payment Required. | Enable billing in Cloud Console. If on Free Tier, you hit the RPD limit (50-1500 depending on model). |
| **403** | Permission Denied. | 1. Check API Key. <br>2. Vertex AI: Grant `roles/aiplatform.user`. <br>3. Check `location` matches project quota (e.g., `us-central1`). |
| **429** | Rate Limit Exceeded. | Implement Exponential Backoff (1s, 2s, 4s). Differentiate between RPM (rate) and RPD (quota). |
| **Auth Error** | `auth/api-key-not-valid` (Firebase). | Enable "Identity Toolkit API" in Google Cloud Console. Check API Key restrictions. |

## 6. Code Examples (Unified SDK)

### Python: Gemini 3 with Thinking & Search Grounding

```python
from google import genai
from google.genai import types

# Initialize (Vertex AI or AI Studio)
client = genai.Client(vertexai=True, project="YOUR_PROJECT", location="us-central1")

response = client.models.generate_content(
    model="gemini-3-pro-preview",
    contents="Explain the architectural differences between Veo 3.1 and Sora.",
    config=types.GenerateContentConfig(
        thinking_config=types.ThinkingConfig(thinking_level="high"),
        tools=[types.Tool(google_search=types.GoogleSearch())], # Grounding
        temperature=1.0 # MUST be 1.0
    ),
)

# Print thought process (if available) and final response
for part in response.candidates.content.parts:
    if part.thought:
        print(f"Thought: {part.text}")
    else:
        print(f"Response: {part.text}")
```

```python
            "thoughtSignature": "CiQyYj...<encrypted_string>..." # CRITICAL
        }
    ]},

```

### Python: Veo 3.1 Video Generation

```python
# Generating a video with Veo 3.1
response = client.models.predict_long_running(
    model="veo-3.1-generate-preview",
    contents=[
        types.Part.from_text("Cinematic drone shot of a futuristic cyberpunk city, neon lights, rain.")
    ],
    config=types.GenerateVideoConfig(
        aspect_ratio="16:9",
        resolution="1080p", 
        duration_seconds=6,
        person_generation="allow_adult" 
    )
)
# Note: This returns an Operation. You must poll the operation to get the video URI.
```
