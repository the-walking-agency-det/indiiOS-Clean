## 2024-05-24 - [Fix React.memo breaks with inline arrow functions]
**Learning:** `React.memo` performs a shallow comparison of props to prevent unnecessary re-renders. When a parent component passes an inline arrow function (e.g., `const handleSelect = (id) => {...}`) to a memoized child component, the shallow equality check always fails because a new function reference is created on every parent render. This completely defeats the purpose of `React.memo`, as demonstrated in `ConversationHistoryList.tsx` where history items were re-rendering unnecessarily.
**Action:** Always wrap callback functions passed to memoized children in `useCallback` to ensure stable references across renders, allowing `React.memo` to function as intended.
